Development Principles

This pack is intended to be somewhere between a "french vanilla" pack and a standard tech modpack. I am open to mod suggestions, but I do have some ground rules:

  1. No deprecating vanilla Mods should not make a vanilla mechanic entirely pointless. Some finer points:
    • Tinkers' Construct's repair mechanic makes vanilla anvils (and by extension all vanilla tools) pointless. There's no reason to use up anvil durability, materials, and xp levels when one could use an equivalent Tinkers' tool and repair using only materials.
    • Journeymap (and related minimap plugins) make vanilla maps useless; but in this case I am still undecided. Vanilla map mechanics are particularly bad.
    • Waypoint features (also part of journeymap) make vanilla beacons free, but the waypoint features are so useful (particularly on a multiplayer server) that perhaps instead we should consider Minecraft to have balanced this feature too heavily end-game.
    • Ore doubling does not count. Mods use more ores than vanilla, so it is appropriate to have ore "go further". Additionally, since Mojang updated ore drops in 1.7 (now raw forms instead of the blocks), the base game supports a form of ore multiplication (via fortune enchants).
  2. No changing worldgen A world generated in vanilla and in the modpack should mostly be the same. Additional world decorations (e.g. Botania flowers) and ores are ok, but not new biomes/structures/etc.
  3. No duplicate functionality There's no need for Thermal Expansion and Mekanism and IndustrialCraft, etc. I want the modpack to be small and performant. Though since mods often have enormous scope, this guideline can be bent a bit if a mod provides a fun feature unique to it.
